
# 🏰 랑그릿사 부유도시 계약 최적배치 시뮬레이터

이 프로젝트는 모바일 게임 **'랑그릿사 모바일'**의 부유도시 시스템 중 하나인 **[계약]** 퍼즐 콘텐츠를 위한 최적화 도구입니다. 복잡한 진영 간 상성과 퍼즐별 고유 연결망을 계산하여 최대 보너스 점수를 획득할 수 있는 배치를 자동으로 제안합니다.

---

## 💡 주요 기능

1.  **보유 영웅 관리**: 230명 이상의 영웅 데이터베이스에서 내가 실제로 보유한 영웅들을 선택하여 풀(Pool)을 구성합니다.
2.  **영웅 도감 (Database)**: 모든 영웅의 진영과 클래스 정보를 한눈에 확인하고, 진영별 필터링을 통해 전략적으로 멤버를 구성할 수 있습니다.
3.  **4종 퍼즐 동시 최적화**: 시청, 산림연맹, 연구원, 형제회 등 서로 다른 구조를 가진 **4가지 계약지에 대해 보유 영웅을 가장 효율적으로 나누어 배치**하는 기능을 제공합니다.
4.  **그래프 기반 최적화 알고리즘**: 단순 그리드가 아닌, 각 노드(영웅) 간의 복잡한 연결 선(Edge)을 분석하여 전체 계약지의 총합 점수를 극대화합니다.
5.  **실시간 스코어 시뮬레이션**: 영웅 배치 변경 시 각 계약지별 점수와 전체 합산 점수를 즉각적으로 시각화합니다.
6.  **데이터 자동 저장**: 브라우저의 Local Storage를 활용하여 선택한 영웅 풀과 클래스 설정, 배치 상태를 자동으로 저장합니다.

## ⚖️ 배치 및 점수 규칙

사용자는 보유한 영웅들 중 정예 멤버를 선택합니다. 시스템은 이 영웅들을 **4개의 구역(총 28개 노드)에 중복 없이 분산 배치**하여 전체 보상 합계를 최대로 만듭니다.

### 🎯 점수 계산 공식 (진영 시너지)
- **기본 점수**: 배치된 영웅 한 명당 1점
- **시너지 점수**: 연결된 두 노드 사이의 진영 중첩 개수에 따라 추가 점수 부여
  - 진영 1개 겹침 시: +1점
  - 진영 2개 겹침 시: +2점
  - 진영 3개 겹침 시: +3점 (시각적으로 굵은 파란색 선으로 강조됨)

### 🚫 배치 제한 사항
- **클래스 매칭**: 각 퍼즐 슬롯은 지정된 클래스를 보유한 영웅만 배치할 수 있습니다.
- **중복 금지**: 한 영웅은 여러 계약지에 동시에 배치될 수 없습니다.

## 🧐 개발 기타 의견: 자동화 vs 사람

**“이 퍼즐에서 컴퓨터 자동화가 항상 사람보다 높은 점수를 낼 수 있을까?”**

결론은 **가능은 하지만, 항상 그렇지는 않습니다.**

이 퍼즐은 가능한 모든 조합을 계산하는 '전수 탐색'이 현실적인 시간 내에 불가능한 고난도 조합 최적화 문제(NP-Hard 성격)입니다. 따라서 컴퓨터 알고리즘 역시 지역 최적해(Local Optima)를 찾기 위한 휴리스틱(Heuristic)과 반복적 교환 알고리즘에 의존하게 됩니다.

반면, 사람은 퍼즐의 기하학적 구조와 특정 핵심 영웅(예: 다중 진영 보유자)의 가치에 대한 **직관과 경험**을 통해 초기 전략 수립 단계에서 강력한 강점을 가집니다.

이에 본 프로젝트는 **사람의 판단(영웅 풀 선별 및 클래스 지정)과 컴퓨터의 계산(복잡한 연결망 대조)**을 결합한 하이브리드 접근을 지향합니다. 
현재는 구현되지 않았지만 향후 사용자가 일부 배치를 직접 고정(Pin)한 뒤 나머지 슬롯에 대해서만 자동 최적화를 수행하는 방향으로 기능을 확장할 예정입니다.
